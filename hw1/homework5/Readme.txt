Реализуйте новую версию сервиса сокращения URL,
который хранит соответствия ключей и URL в базе данных.

Конфигурация (settings.py) и urlpatterns перемещены в
соответствующие файлы в проекте, а логика сокращения Url теперь
будет находиться в отдельном приложении url_shortener:

- функции index и redirect_view в файле views.py
- модель Url в models.py
- шаблон в templates/index.html
- тесты в tests.py

В прошлой реализации мы не проверяли, хранится ли уже
такая ссылка в кеше, и просто генерировали новый ключ.
В новой реализации, если ссылка уже сохранена - нужно
вернуть ее существующий ключ.

Также вам надо реализовать счетчик редиректов - redirect_count,
который будет увеличиваться при каждом переходе по ссылке.

Подробные требования к модели смотрите в файле models.py

Для тестирования запустите

./manage.py test

Для запуска сервера для разработки

./manage.py runserver

Если не используете sqlite - отредактируйте раздел
DATABASES в settings.py и пропишите настройки вашей БД.
